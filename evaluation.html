Participants in NLPContributionGraph may participate in any or all of the 3 tasks (Sentence, Entity, Triples). The evaluation metrics that will be applied are:

  • Sentence: precision, recall and F1
  • Entity: precision, recall and F1
  • Triples: precision, recall and F1 overall and for each attribute

The focus of NLPContributionGraph is on shallow semantic structuring of the contributions in NLP scholarly articles to form a knowledge graph. To allow a thorough evaluation of systems, NLPContributionGraph will have multiple evaluation phases:

  1. Practice phase:
    • Training and dev data:
      • contribution sentences from NLP scholarly articles across different IE tasks (e.g., machine translation, named entity recognition, etc.)
      • entities from the annotated contribution sentences
      • triples from the entities under any of the fifteen information units listed below
        • ResearchProblem, Approach, Model, Method, Architecture, System, Application, Objective, ExperimentalSetup, Hyperparameters, Results, Tasks, Experiments, AblationAnalysis, and Baselines
    • Test data:
      • In the pratice phase, the test data is the dev set itself with a different set of annotated articles from those chosen as training data
  2. End-to-end pipeline testing phase:
    • Training and dev data:
      • Same as in the practice phase; no new data will be released
    • Test data:
      • Participants will be provided with unannotated articles that are not part of the training or dev set
      • The server will host the reference annotations for these articles for participant automated system output evaluations
  3. Entity and Schema extraction testing phase:
    This phase will have two parts: In part 1, the participant systems will be given the gold annotated sentences and will be expected to provide purely their entity extraction output; In part 2, the participant systems will be given the gold entities and will be expected to provide their system output just for triples.
    • Training data:
      • Same as in the practice phase; no new data will be released
    • Test data:
      • Part 1: Gold annotated contribution sentences
      • Part 2: Gold annotated entities from the contribution sentences

System Output Format

Based on the subtask, the format of submissions is the same for both phase 1 and phase 2. Regardless of the train, dev, or test datasets, the participants will be provided the articles in plain text format and lightly preprocessed in terms of sentence splits and tokenization. For the Sentence task, sentence numbers, with a start counter at 1, will be the input data to participating systems and are expected to produce their predictions similarly. For the Entity task, the first and last token numbers, where tokens also have a start counter of 1 for each sentence, will be the input and expected output from participants. And finally, for the Triples task, triples of the entities as their token numbers, thus six numbers per line, will be the input data and expected output data from the participant systems. The submission directory structure should look like:

    • Sentence.txt
    • Entity.txt
    • Triples.txt
  • ArticleID02
    • Sentence.txt
    • Entity.txt
